---

## 1. 当前产品定位

GitNovelBox 是一个：

- 纯前端静态部署
- 以 GitHub 仓库作为存储后端
- 面向 TXT 小说与轻量文本资料管理
- 支持浏览、搜索、预览、同步、统计的网页网盘

## 4. 建议优先级

### P0：最值得优先补的功能

这些功能最直接提升“像网盘一样可用”的体验：

1. **拖拽上传**
2. **多选 + 批量操作**
3. **页面打开期间定时同步**
4. **全文搜索**
5. **更明显的同步任务状态条**

### P1：次优先级

这些功能能显著提升长期使用体验：

1. 阅读书签
2. 最近阅读
3. Token 加密存储
4. IndexedDB 缓存
5. 多仓库切换

### P2：中后期增强

这些功能更适合作为成熟版本扩展：

1. 章节识别
2. 标签体系
3. 分类视图
4. 快捷键体系
5. 更丰富的图表统计

## 5. 推荐版本路线图

### v0.1 当前基线版

目标：先把基本可用的静态网盘跑通。

- GitHub 连接
- 本地目录扫描
- 远端树展示
- 单文件/一键同步
- 文件预览
- 搜索 / 排序 / 统计
- 右键菜单
- 面包屑导航
- 列表 / 卡片切换

### v0.2 易用性增强版

目标：让它更像真正的网盘。

- 拖拽上传
- 多选与批量操作
- 页面打开期间自动同步
- 更明显的同步进度条
- 最近操作记录
- 高级筛选

### v0.3 阅读与检索增强版

目标：让它更适合长期阅读与整理。

- 全文搜索
- 阅读书签
- 阅读进度
- 章节识别
- 分类与标签

### v1.0 稳定发布版

目标：从“项目原型”变成“可长期自用工具”。

- IndexedDB 本地缓存
- Token 加密存储
- 多仓库管理
- 更稳定的大目录性能
- 更完整的异常恢复
- 更成熟的界面和交互细节

## 7. 最终目标形态

GitNovelBox 的理想形态，不是变成一个通用大网盘，而是：

> **一个无需服务器、基于 GitHub 私有仓库、适合 TXT 小说和轻量文本资料管理的可视化个人网盘。**

它应该具备：

- 足够像网盘
- 足够轻量
- 足够容易部署
- 足够适合个人长期自用

---

## 8. 一句话总结

当前版本已经是一个 **能部署、能浏览、能搜索、能预览、能同步、能看统计** 的静态网页网盘原型；后续最值得投入的是 **批量操作、自动同步、全文搜索和阅读体验**。
